Quick start

처음 쓸 때 보는 순서

1. 메인 URL 등록

`https://storybeing.tistory.com` 같은 공개 블로그 메인 주소를 넣습니다.

2. Discover

RSS, sitemap, wp-json, 메인 링크 순으로 공개 글 URL을 찾습니다.

3. Analyze Now

algorithm이 게시글별 qualityScore와 개선 제안을 계산합니다.

4. 상세 확인

낮은 점수 글부터 보고 바로 수정 우선순위를 정합니다.

점수 01

qualityScore

대표 점수는 게시글 단위 qualityScore 0~100입니다.

점수 02

5개 구성 항목

제목/훅, 가독성, 정보 가치, 차별성, 검색 적합성 평균으로 계산합니다.

점수 03

왜 기존 종합점수를 뺐나

설명이 어려운 내부 종합지표보다 글마다 바로 이해되는 점수를 우선하기 위해서입니다.

Storybeing

`53개`가 왜 `14개`가 되었나

예전 문제

티스토리 메인 링크에서 `tag`, `archive`, `category`가 게시글로 잘못 들어가던 버그가 있었습니다.

현재 기준

`storybeing.tistory.com`은 `rss 10 / sitemap 4 / main 0 / wp-json 0`, 총 14개가 정상 결과입니다.

현재 규칙

티스토리는 숫자형 URL과 `/entry/`만 게시글로 인정합니다.

Policy

주의사항

공개 페이지만 대상

로그인, 비공개, 권한 우회가 필요한 영역은 수집하지 않습니다.

네이버는 opt-in

네이버는 자동 수집 정책 리스크 때문에 기본 비활성 상태이며, 설정에서 직접 켜야 합니다.

LLM은 선택 사항

API 키가 없어도 전체 수집/분석/대시보드 사용이 가능합니다. AI는 문장 보강만 담당합니다.
