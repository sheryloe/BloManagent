Quick start

처음 사용할 때 보는 순서

앱 안의 도움말 페이지와 동일한 흐름으로, URL 등록부터 분석 확인까지 빠르게 정리했습니다.

1. 메인 URL 등록

예시: `https://example.tistory.com`, `https://blog.naver.com/example`, `https://example.blogspot.com`

2. Discover 실행

공개 RSS, sitemap, wp-json, 메인 링크를 순서대로 확인하며 포스트를 찾습니다.

3. Analyze Now 실행

최근 7일, 최근 30일, 신규 글만 등 범위를 선택해 분석 결과와 추천 액션을 만듭니다.

4. 대시보드 확인

블로그별 EBI, 마지막 수집 시각, 최근 추천 액션, 실행 기록을 함께 보며 우선순위를 정합니다.

Collection

수집 방식

입력은 메인 URL 하나지만, 실제 내부에서는 플랫폼별 공개 구조를 순차적으로 확인합니다.

STEP 1 RSS / Atom 우선

공개 피드가 가장 안정적이기 때문에 먼저 확인합니다. 네이버 블로그는 blogId를 해석해 RSS를 우선 시도합니다.

STEP 2 sitemap 보조 확인

RSS에 없는 글이 sitemap에 남아 있는 경우가 있어 누락 보완용으로 함께 확인합니다.

STEP 3 WordPress는 wp-json 우선

공개 REST API가 있으면 페이지네이션으로 공개 글을 모으고, 없으면 RSS와 sitemap으로 돌아갑니다.

STEP 4 메인 링크 fallback

RSS와 sitemap이 부족할 때만 메인 페이지 링크를 확인해 공개 포스트 후보를 보강합니다.

지원 01

티스토리

`https://example.tistory.com` 형식의 메인 URL을 넣으면 RSS, sitemap, 공개 본문을 확인합니다.

지원 02

네이버 블로그

`https://blog.naver.com/example` 형식의 홈 주소를 기준으로 blogId를 해석해 공개 RSS를 우선 사용합니다.

지원 03

Blogger / WordPress

Blogger는 공개 피드와 본문 HTML, WordPress는 wp-json과 RSS/sitemap 기반으로 접근합니다.

Safe use

안전하게 사용하는 기준

공개 페이지만 대상

로그인 우회나 비공개 글 접근은 하지 않습니다. 이 프로젝트는 공개 URL을 읽는 분석 도구를 목표로 합니다.

서비스 정책 확인

공개 페이지라도 자동 접근 정책은 서비스별로 다를 수 있으니, 배포나 호스팅 전에 robots.txt와 약관을 확인하는 쪽이 안전합니다.

네이버는 특히 주의

네이버 서비스 이용약관에는 사전 허락 없는 자동화 수단으로 회원 ID나 게시물 등을 수집하는 행위를 금지하는 문구가 있어, 실제 사용 전에 정책 검토가 필요합니다.

원문 재배포보다 분석 중심

전체 원문을 다시 공개하기보다 내부 분석 메모, 요약, 개선 포인트를 중심으로 활용하는 것이 더 안전합니다.

FAQ

도움말

RSS URL이 없어도 등록할 수 있나요?

네. 메인 URL만으로 시작할 수 있고, RSS URL은 필요한 경우에만 고급 옵션으로 넣으면 됩니다.

좋아요나 댓글 수는 항상 수집되나요?

아닙니다. 공개 페이지에 실제로 노출된 값만 읽기 때문에 플랫폼이나 스킨 구조에 따라 비어 있을 수 있습니다.
