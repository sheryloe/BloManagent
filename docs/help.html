Quick Start

실행 순서

01

주소 입력

블로그 메인 URL 또는 게시글 URL을 입력합니다.

02

수집 실행

RSS, sitemap, wp-json, HTML로 후보를 모으고 verified와 excluded로 나눕니다.

03

분석 실행

게시글별 등급, 근거, 개선 작업을 생성합니다.

04

초기화

작업이 끝나면 워크스페이스를 비웁니다.

Grade

S~F 등급 구조

화면은 등급 중심으로 보여주고 내부에서는 qualityScore를 계산합니다.

상위 축

  • headline: 제목과 첫인상
  • readability: 문단, 목록, 스캔성
  • value: 예시, 체크리스트, 실행 순서
  • originality: 경험, 비교, 차별 포인트
  • search fit: 질문형 구조, FAQ, 키워드 정렬

등급 컷

  • S: 90 이상
  • A: 80 이상
  • B: 65 이상
  • C: 55 이상
  • D: 45 이상
  • F: 45 미만

Evidence

왜 낮은지 같이 보여줍니다

약한 신호

부족한 축을 등급과 함께 노출합니다.

근거

질문형 문장 수, 예시 표현 미검출 같은 근거를 함께 보여줍니다.

개선 작업

약한 신호 조합에 맞춰 글 내용 기반 액션을 생성합니다.

Tistory

티스토리 검증 규칙

Gate 1

URL 규칙

/{숫자}, /entry/...만 통과시키고 나머지는 제외합니다.

Gate 2

실제 페이지 확인

후보 URL을 다시 fetch해 article marker를 확인합니다.

Gate 3

verified / excluded

애매한 URL은 excluded로 보관하고 집계에서는 제외합니다.

Case

storybeing 기준

현재 기준 verified posts 14개, `rss 10 / sitemap 4 / main 0`입니다.

Policy

정책 범위

항목 현재 처리 메모
비공개 글 수집 안 함 로그인과 권한이 필요한 영역은 다루지 않습니다.
관리자 통계 수집 안 함 조회수, 공감, 댓글도 공개 페이지 기준으로만 시도합니다.
Naver Blog 기본 비활성 opt-in이 없으면 차단합니다.
AI 엔진 선택 사항 서술 보강만 담당합니다.

Recovery

복구 기준

1. 저장소

명세서.md와 README.md를 먼저 확인합니다.

2. wiki

운영 원칙과 아키텍처를 wiki에서 확인합니다.

3. Step 원고

docs/notion/BloManagent와 노션 Step을 기준으로 이어갑니다.
