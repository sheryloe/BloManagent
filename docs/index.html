Executive Overview

공개 블로그를 수집하고, 검증하고, 게시글 단위로 진단하는 로컬 분석 작업대

BloManagent는 메인 URL 또는 게시글 URL 하나로 공개 블로그를 수집하고, 검증된 게시글만 대상으로 S~F 등급과 신호별 근거, 개선 작업을 보여주는 휘발성 분석 워크스페이스입니다.

Summary

현재 기준 핵심 요약

입력 기준 메인 URL / 게시글 URL

게시글 주소를 입력해도 가능한 범위에서 블로그 기준으로 정규화합니다.

검증 기준 Verified posts only

집계와 분석에는 검증된 공개 게시글만 포함합니다.

출력 기준 S ~ F grade

내부 qualityScore를 계산한 뒤 화면에는 등급 중심으로 보여줍니다.

운영 기준 휘발성 워크스페이스

리포트는 최신 스냅샷만 유지하고, 필요 시 전체 초기화가 가능합니다.

Pipeline

수집과 검증 흐름

공개 데이터만 수집하고, 검증된 게시글만 집계와 분석 대상으로 올립니다.

01

Discovery

RSS, sitemap, WordPress wp-json, HTML 링크 fallback 순서로 후보 URL을 찾습니다.

02

Verification

URL 규칙, 실제 fetch 결과, article marker를 기준으로 verified 또는 excluded로 분리합니다.

03

Analysis

게시글별 신호와 근거를 추출하고 5개 축 기준으로 qualityScore를 계산합니다.

04

Reporting

최신 리포트 한 건만 유지하면서 Best/Worst, 분포, 병목, 실행 로그를 보여줍니다.

티스토리 검증 기준

티스토리는 RSS 우선, sitemap 보완, main fallback 최후 순서로 동작합니다. `category`, `tag`, `archive`, `guestbook` 류 URL은 집계 대상에서 제외합니다.

Analysis Model

분석 모델과 리포트 구조

5개 상위 축

  • headline: 제목과 첫인상
  • readability: 문단, 소제목, 스캔성
  • value: 예시, 실행 순서, 실전성
  • originality: 경험 신호와 차별성
  • search fit: 검색 의도와 FAQ 적합성

리포트에 보이는 항목

  • 전체 등급 분포와 범위
  • Best / Worst 게시글 제목
  • 강한 신호 / 약한 신호
  • 신호별 근거와 개선 작업
  • 최신 실행 로그와 권장 액션
