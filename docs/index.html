Report Cover

공개 블로그 진단 리포트 워크스페이스

메인 URL 또는 게시글 URL 하나로 공개 블로그를 수집하고, 검증된 게시글만 대상으로 S~F 등급, 근거, 개선 작업을 보여줍니다.

저장소 보기 운영 매뉴얼
Scoring Algorithm fixed

등급 계산은 algorithm이 고정으로 맡습니다.

Workspace Resettable

필요할 때 수집하고 끝나면 비웁니다.

Summary

현재 기준

입력 메인 URL / 게시글 URL

게시글 주소도 블로그 기준으로 정규화합니다.

집계 Verified posts only

집계와 분석에는 verified post만 포함합니다.

출력 S ~ F grade

내부 qualityScore를 등급으로 변환해 보여줍니다.

운영 휘발성 워크스페이스

최신 스냅샷만 유지하고 전체 초기화가 가능합니다.

Pipeline

수집과 검증

공개 데이터만 읽고 verified post만 분석 대상으로 올립니다.

01

Discovery

RSS, sitemap, wp-json, HTML fallback 순서로 후보 URL을 찾습니다.

02

Verification

URL 규칙과 article marker 기준으로 verified와 excluded를 분리합니다.

03

Analysis

신호와 근거를 추출하고 5개 축 기준으로 qualityScore를 계산합니다.

04

Reporting

최신 리포트 한 건만 유지하고 Best/Worst와 분포를 보여줍니다.

티스토리 기준

RSS 우선, sitemap 보완, main fallback 최후 순서로 동작합니다. `category`, `tag`, `archive`, `guestbook` 류 URL은 집계 대상에서 제외합니다.

Analysis

분석 모델

5개 상위 축

  • headline: 제목과 첫인상
  • readability: 문단과 스캔성
  • value: 예시와 실행 순서
  • originality: 경험과 차별성
  • search fit: 검색 의도와 FAQ

출력 항목

  • 전체 등급 분포
  • Best / Worst 게시글 제목
  • 강한 신호 / 약한 신호
  • 신호별 근거와 개선 작업
  • 실행 로그와 권장 액션
