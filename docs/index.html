BloManagent는 여러 블로그를 동시에 운영할 때 흩어지기 쉬운 변화 추이와 게시물 흐름을 로컬 퍼스트 구조 안에서 비교하고 해석하기 위해 만든 프로젝트입니다. 단순한 조회수 패널이 아니라, 수집기와 SQLite 스키마, AI 분석 파이프라인, 대시보드가 어떻게 이어지는지를 블로그 글처럼 설명하는 페이지로 바꿨습니다.

수집, 비교, 해석이 하나의 분석 루프로 이어집니다

데이터를 모으는 단계와 AI 해석 단계를 분리하지 않고, 운영자가 다시 돌아와 읽기 좋은 문서형 페이지로 정리했습니다.

01

기획 배경과 로컬 퍼스트 아키텍처

블로그 분석 이력을 누적하고 주간 단위로 비교하기 쉬운 구조를 먼저 설계했습니다.

02

SQLite 스키마와 수집기 중심 설계

분석 품질보다 먼저 데이터 축적 구조를 명확히 해 반복 수집과 비교가 쉬워지도록 했습니다.

03

AI 분석 파이프라인과 대시보드 연결

수집 데이터, AI 해석, 주간 비교 결과를 한 페이지 서사처럼 읽히도록 공개용 페이지를 구성했습니다.

Chapters

Notion 문서 스텝 구조를 따라가면 이 저장소가 단순 분석 툴이 아니라 장기 운영을 위한 비교 관찰 도구라는 점이 자연스럽게 드러납니다.

1. 기획 배경과 로컬 퍼스트 아키텍처 설계

왜 분석 데이터를 가까이에 두는 구조를 먼저 고정했는지 설명합니다.

2. SQLite 스키마와 블로그 수집기 구현 과정

비교 가능한 데이터를 꾸준히 쌓는 것이 기능 추가보다 먼저라는 판단이 드러나는 단계입니다.

3. AI 분석 파이프라인과 대시보드 구현 과정

수집된 데이터를 해석 가능한 인사이트로 바꾸는 흐름을 설명합니다.

4. 테스트, 빌드, Windows 실행, 향후 확장 전략

운영형 저장소답게 실행 환경과 다음 단계 확장 포인트까지 공개합니다.

Analysis notes

분석 페이지가 강조하는 실제 운영 포인트

스크린샷보다 먼저 드러나야 하는 수집-비교-해석 루프를 카드 형태로 정리해, 검색 유입 사용자가 구조를 빠르게 이해할 수 있게 했습니다.

STEP 1 메인 URL을 등록하고 수집 주기를 정합니다

여러 블로그의 기준 URL을 같은 구조로 관리해 주간 비교가 가능하게 만듭니다.

STEP 2 AI 해석과 대시보드 비교를 함께 봅니다

수치만 보는 것이 아니라 변화 원인과 운영 해석을 같이 읽는 흐름이 핵심입니다.

Quick start

`npm install` 후 `npm run dev`, 배포 전에는 `npm run build`와 `npm run start`를 사용합니다.

Repository structure

`apps/`, `packages/`, `docs/`로 나누어 서버와 웹, 공용 로직, 문서를 구분합니다.

Audience

여러 블로그를 동시에 운영하며 로그 기반으로 추세를 비교하고 싶은 사용자에게 맞춘 구조입니다.

Search intent

블로그 분석 검색 의도에 맞춘 문장과 구조

국문 키워드

블로그 분석 대시보드, 로컬 퍼스트 블로그 분석, 주간 비교 분석 같은 검색어를 본문 안에 직접 녹였습니다.

FAQ

자주 묻는 질문

BloManagent는 무엇을 분석하나요?

티스토리, 네이버 블로그, Blogger 계열 블로그의 변화 추이와 게시물 흐름을 주간 단위로 분석합니다.

왜 로컬 퍼스트 구조인가요?

분석 이력과 운영 데이터를 PC 또는 내부 환경에 가깝게 두어 반복 비교와 추적을 편하게 하기 위해서입니다.

AI 분석은 어떤 옵션을 고려하나요?

OpenAI, Google AI Studio, Ollama 같은 옵션을 고려한 구조입니다.
