# BloManagent 노션 페이지 구조

이 문서는 노션 상위 페이지와 하위 Step 1~5 문서를 어떤 구조로 배치하면 좋은지 정리한 안내서입니다.  
이번 시리즈는 단순 개발 일지가 아니라, 티스토리에 바로 올릴 수 있는 SEO형 구현기 시리즈를 목표로 작성했습니다.

## 상위 페이지 제목 제안

`BloManagent | 메인 URL 하나로 시작한 공개 블로그 진단 도구 제작기`

## 상위 페이지 소개 문단

BloManagent는 메인 URL만 입력하면 공개 블로그를 수집하고, 게시글별 `qualityScore`와 개선 포인트를 보여주는 로컬 중심 분석 워크스페이스입니다.  
이 시리즈는 기획, 데이터 구조, 설명 가능한 분석 알고리즘, 대시보드 UX, 티스토리 strict filtering까지 실제 구현 과정을 Step 1~5로 정리합니다.

## 하위 페이지 구성

- Step 1. 메인 URL 기반 블로그 진단 도구를 기획한 이유
- Step 2. SQLite와 공개 피드 기반 블로그 수집기 설계
- Step 3. LLM 없이 qualityScore를 계산하는 설명 가능한 알고리즘
- Step 4. 대시보드와 GitHub Pages 문서를 제품처럼 다듬는 과정
- Step 5. 티스토리 sitemap 오탐을 막는 strict verified-post discovery

## 티스토리 게시용 운영 팁

- 각 Step는 제목, 메타 설명, 슬러그, 핵심 키워드를 먼저 넣었습니다.
- 본문은 `문제 -> 설계 -> 구현 -> 결과 -> 다음 Step` 순서로 읽히게 구성했습니다.
- 시리즈 글끼리 내부 링크를 강하게 연결하면 체류시간과 회차형 유입에 유리합니다.
- Step 5에는 `storybeing.tistory.com` 사례를 직접 넣어 실제 수정 로그가 드러나게 했습니다.

## 추천 게시 순서

1. Step 1로 문제 정의와 제품 컨셉을 먼저 공개합니다.
2. Step 2와 Step 3에서 구현 핵심인 수집과 점수 모델을 붙입니다.
3. Step 4에서 문서와 UX를 다뤄 프로젝트 완성도를 보여줍니다.
4. Step 5에서 티스토리 오탐 수정 사례를 다뤄 검색형 유입을 노립니다.

## 시리즈 전체 CTA 문구 예시
