# BloManagent 노션 페이지 구조

이 문서는 BloManagent 시리즈를 노션 상위 페이지와 하위 Step 문서로 어떻게 배치할지 정리한 안내문입니다.
시리즈 목표는 단순 개발 일지가 아니라, 티스토리나 워드프레스 블로그에 그대로 옮겨 적을 수 있는 SEO 구조의 구현 기록을 만드는 것입니다.

## 상위 페이지 제목

`BloManagent 관련 글 아카이브`

## 상위 페이지 설명

BloManagent는 메인 URL 하나로 공개 블로그를 수집하고, 게시글별 등급과 개선 포인트를 보여주는 분석 워크스페이스입니다.
이 시리즈는 아이디어, 수집기 설계, qualityScore 알고리즘, 대시보드 UX, 티스토리 strict filtering, 모델별 분석 신뢰도까지 Step 1~6으로 정리합니다.

## 하위 페이지 구성

- Step 1. 메인 URL 기반 블로그 진단 도구를 기획한 이유
- Step 2. SQLite와 공개 피드 기반 블로그 수집기 설계
- Step 3. LLM 없이 qualityScore를 계산하는 설명 가능한 알고리즘
- Step 4. 대시보드와 GitHub Pages 문서를 제품처럼 다듬는 과정
- Step 5. 티스토리 sitemap 오탐을 막는 strict verified-post discovery
- Step 6. 모델별 분석 방법과 분석 신뢰도를 어떻게 분리했는가

## 시리즈 운영 기준

- 모든 글은 UTF-8 한글 기준으로 작성합니다.
- 각 글은 2,000~3,000자 범위를 유지합니다.
- 상단에는 `SEO 패키지`를 둡니다.
- 본문에는 티스토리 복붙용 `HTML 구조 예시`를 넣습니다.
- 문장 흐름은 `문제 -> 설계 -> 구현 -> 결과 -> 다음 글 연결` 순서를 유지합니다.

## 권장 게시 순서

1. Step 1에서 문제 정의와 제품 방향을 설명합니다.
2. Step 2에서 수집기와 저장 구조를 설명합니다.
3. Step 3에서 알고리즘 기반 qualityScore를 설명합니다.
4. Step 4에서 대시보드, README, GitHub Pages를 제품형으로 정리한 이유를 설명합니다.
5. Step 5에서 티스토리 false positive를 막은 strict verified-post discovery를 설명합니다.
6. Step 6에서 algorithm, OpenAI, Google, Ollama의 역할과 신뢰도를 설명합니다.
